# lab-report-formatter

Python-скрипт для автоматического форматирования `.docx`-отчёта по требованиям лабораторной работы.

Программа приводит документ к заданному оформлению, форматирует основной текст, заголовки, таблицы и рисунки, а также добавляет или перенумеровывает подписи вида:

- `Таблица N — ...`
- `Рисунок N — ...`

После тире по умолчанию ставится `...`, чтобы пользователь мог вручную вписать название.

---

## Что делает программа

### Параметры страницы
- формат листа: **A4**;
- поля: верхнее и нижнее — **2 см**, левое — **3 см**, правое — **1 см**.

### Колонтитулы
- верхний колонтитул: **по левому краю**, `ФИО, группа, Отчет по лабораторной работе N`;
- шрифт верхнего колонтитула: **Times New Roman, 12 pt, курсив, черный**;
- положение верхнего колонтитула относительно верхнего края: **0,9 см**;
- нижний колонтитул: **сквозная нумерация страниц**;
- номер страницы: **по центру**, **Times New Roman, 12 pt, обычный, черный**.

### Основной текст
- **Times New Roman, 14 pt, черный**;
- выравнивание **по ширине**;
- отступ первой строки **1,25 см**;
- межстрочный интервал **1,5**;
- без дополнительных интервалов до/после абзаца;
- включается запрет висячих строк;
- отключаются автоматические переносы;
- убираются мягкие переносы.

### Заголовки разделов
Программа форматирует только те заголовки, которые в Word помечены стилем:

- `Heading 1`, или
- `Заголовок 1`

Для таких заголовков применяется:
- сквозная нумерация разделов;
- формат `N ТЕКСТ ЗАГОЛОВКА` без точки после номера и в конце;
- начало каждого раздела **с новой страницы**;
- выравнивание **по левому краю**;
- левый отступ **1,25 см**;
- межстрочный интервал **1,5**;
- отступ после **6 pt**;
- шрифт **Arial, 14 pt, bold, uppercase**.

### Таблицы
- таблица выравнивается по центру;
- для строк таблицы включается запрет разрыва строки между страницами;
- автоматическое повторение заголовочной строки на следующей странице **отключается**;
- первая строка оформляется как головная:
  - **Times New Roman, 12 pt, bold, по центру**;
- остальной текст таблицы:
  - **Times New Roman, 12 pt, обычный, по левому краю**;
- подпись таблицы ставится **перед таблицей**;
- подписи таблиц автоматически добавляются или перенумеровываются по шаблону:
  - `Таблица N — ...`
- в GUI доступна опция **автоматической подгонки ширины столбцов таблиц**.

### Рисунки
- абзац с рисунком центрируется;
- подпись рисунка ставится **после рисунка**;
- подписи рисунков автоматически добавляются или перенумеровываются по шаблону:
  - `Рисунок N — ...`
- стиль подписи рисунка:
  - **Times New Roman, 12 pt, по центру, интервал 1,5**;
- программа старается сохранять и подписывать рисунки как в обычном тексте, так и **внутри ячеек таблиц**.

### GUI и CLI
Программа поддерживает два режима работы:
- графический интерфейс;
- запуск из командной строки.

В главном окне GUI есть:
- выбор входного `.docx`;
- выбор выходного `.docx`;
- поля для ФИО, группы и номера лабораторной работы;
- опция подгонки ширины столбцов таблиц;
- кликабельная ссылка на страницу курса:
  - [Страница курса / требования](https://online.mospolytech.ru/mod/page/view.php?id=308998)

## Что программа не гарантирует

Программа **не гарантирует на 100% автоматически**:

1. **Логичную ссылку на рисунок или таблицу в предыдущем абзаце.**  
   Скрипт умеет добавлять и перенумеровывать подписи, но не пишет за автора фразы вроде `на рисунке 3 показано...` или `в таблице 2 приведены...`.

2. **Идеальное русскоязычное строкоразбиение.**  
   Скрипт не гарантирует, что строка не закончится предлогом, союзом или частицей, а новая строка не начнется со знака препинания.

3. **Идеальную подгонку документа по страницам.**  
   Он не может надежно подогнать текст, рисунки и таблицы так, чтобы внизу каждой страницы оставалось не более одной пустой строки.

4. **Абсолютно безошибочное удержание подписи и объекта на одной странице во всех документах.**  
   Для рисунков и таблиц применяются настройки, снижающие риск разрыва, но поведение Word зависит от содержимого документа, размеров объектов и внутренней вёрстки `.docx`.

5. **Автоматическое исправление всех проблем исходного документа.**  
   Если в документе изначально используются нестандартные стили, сложные секции, текстовые поля, плавающие объекты, вложенные таблицы, особая ручная верстка или экзотическое форматирование, результат может потребовать ручной проверки.

6. **Полную семантическую корректность подписей.**  
   Скрипт вставляет шаблонные подписи вида `Рисунок N — ...` и `Таблица N — ...`, но не определяет содержательное название автоматически.

7. **Полную совместимость со всеми вариантами Word-документов.**  
   Программа рассчитана на обычные `.docx`, но документы после конвертации из PDF, Google Docs, LibreOffice или других редакторов могут содержать нестандартную структуру.

8. **Идеальную работу автоподгона ширины столбцов на всех таблицах.**  
   Эта функция является **эвристической**: в одних таблицах она улучшает вёрстку, в других может сделать её хуже. Поэтому опция сделана отдельной и используется по желанию.

9. **Корректную обработку абсолютно всех типов изображений в старых документах.**  
   Программа умеет работать с обычными изображениями Word (`drawing`) и частью legacy-изображений (`pict / vml`), но документы со сложными объектами и старой вставкой графики всё равно лучше проверять вручную.

Итог: программа **сильно ускоряет подготовку отчёта**, но **не заменяет финальную ручную проверку документа в Microsoft Word**.

## Запуск

### 1. Графический интерфейс

```bash
python LogachevFormatter.py
```

В GUI можно указать:
- входной `.docx`;
- выходной `.docx`;
- ФИО;
- номер группы;
- номер лабораторной работы;
- включить или отключить автоподгон ширины столбцов таблиц.

### 2. Командная строка

```bash
python LogachevFormatter.py input.docx output.docx --fio "Иванов И.И." --group "ИВТ-101" --lab 5
```

Если в вашей версии скрипта поддерживается флаг умной подгонки ширин, запуск может выглядеть так:

```bash
python LogachevFormatter.py input.docx output.docx --fio "Иванов И.И." --group "ИВТ-101" --lab 5 --smart-widths
```

Если выходной файл не указан, программа формирует имя автоматически:

```text
input_formatted.docx
```

## Как программа определяет заголовки разделов

Раздел считается заголовком только в том случае, если абзац в Word уже помечен стилем:

- `Heading 1`
- `Заголовок 1`

То есть структура разделов должна быть размечена в исходном документе заранее.

## Что желательно проверить вручную после обработки

После работы скрипта рекомендуется открыть документ в Word и проверить:
- корректность названий у всех рисунков и таблиц;
- наличие ссылок на рисунки и таблицы в основном тексте;
- переносы строк в длинных заголовках;
- отсутствие лишних пустых мест внизу страниц;
- расположение крупных таблиц и рисунков;
- корректность оформления сложных объектов;
- сохранность изображений, особенно если документ старый или собран из разных редакторов;
- поведение таблиц, если включалась опция автоподгона ширины столбцов.
